## 原型链

当访问对象属性时，如果自身没有该属性，就会沿着**原型对象**逐级向上查找，直到 null 为止。这个查找路径就称为原型链。 

### 原型对象？

Js中，每个函数都有一个 `prototype` 属性，它指向一个对象，所有通过这个构造函数创建的实例都会共享这个对象里的属性和方法  ，这个对象就叫**原型对象**

之所以实例都能共享，是在new一个实例的时候，js做了这样的操作将实例自动连接到 `Person.prototype`

```javascript
p1.__proto__ === Person.prototype
```

### 对象原型?

对象实例的 **proto，**它是 当前对象内部指向原型对象的那个**引用**  ，即对象的 **proto** 指向构造函数的 prototype

```javaScript
function Person(name) {
  this.name = name;
}

const p = new Person("Tom");

原型链为
 p
 ↓
Person.prototype
 ↓
Object.prototype
 ↓
null
```

### constructor?

constructor是原型对象上的属性，用来指向构造函数，它是在原型对象上的，而不是在对象实例上，所以才有

```javascript
p.constrctor = Person
//p上没有constrctor，所有往原型对象上找，找到p.prototype
```

### 原型链继承

```javascript
function Animal(name) {
  this.name = name;
}

Animal.prototype.eat = function () {
  console.log("eat");
};

function Dog(name) {
  this.name = name;
}

Dog.prototype = new Animal();
```

此时

```
dog
 ↓
Dog.prototype
 ↓
Animal.prototype
 ↓
Object.prototype
```

所以

```
const dog = new Dog("旺财");

dog.eat(); // 可以用
```

这就是原型链继承

## 变量提升

函数声明优先于 `var` 声明提升，但赋值不会提升

```
var getName = function () {

  console.log(4);

};

function getName() {

  console.log(5);

}
```

预编译阶段：

```
function getName() {
  console.log(5);
}

var getName; // 只声明
```

执行阶段：

```
var getName = function () {
  console.log(4);
};
```

所以原来的函数(5) 被覆盖



声明变量没有加上写 `var` / `let` / `const`，会被认为是全局变量



## this的指向

普通函数，谁调用指向谁

**箭头函数没有自己的 this**，它会继承 **外层作用域的 this**。  

```
const btn = document.querySelector('.btn')

// 箭头函数 此时 this 指向了 window
btn.addEventListener('click', () => {
  console.log(this)
})
```

如果换成普通函数

```
btn.addEventListener('click', function () {
  console.log(this)
})
```

this指向btn

```
function Foo() {
  getName = function () {
    console.log(1);
  };
  return this;
}
```

this指向windows，因为windows.Foo()

## 改变this指向

1. call()方法

```
fn.call(thisArg, arg1, arg2, ...)
```

2. apply()方法

和 `call()` 一样，立即调用函数并改变 this，不同点参数必须是 数组

使用场景:

Math.max()求数组最大值的时候

需要这样，`Math.max()`**不能直接传数组**

```
Math.max(1, 5, 3, 9, 2)
```

利用扩展运算符可以将数组展开

利用apply的第二个参数是数组的特性，

```
Math.max.apply(null, arr)
```

3. bind()

不会调用函数，返回值是this重新指向好的新函数

而call和apply的返回值是函数调用后的结果值

## 事件循环

执行代码和收集异步任务的模型，在**调用栈**空闲，反复调用**任务队列**里回调函数的执行机制，就叫事件循环

![image.png](/images/js进阶/事件循环.png)

### 为什么需要事件循环？

js是单线程，浏览器是多线程，为了避免js线程阻塞

### js内代码如何运行？

1. js执行同步代码，遇到异步代码交给浏览器环境处理
2. 异步代码有了结果，把回调函数放入任务队列排队
3. 当调用栈空闲，反复调用任务队列里的回调函数

### 异步任务分为微任务和宏任务

之所以有宏任务和微任务的划分，就是为了给异步任务的优先级进行划分，微任务的优先级大于宏任务，微任务全部执行完后才会执行宏任务，因为promis.then是用于数据的获取，当数据获取完就应该马上被处理，而不是跟其他的异步任务一起争夺谁先执行

## 事件流

### 事件委托、事件冒泡、事件捕获有什么关系？  

 DOM 事件传播分为三个阶段：捕获阶段、目标阶段和冒泡阶段。事件捕获是从外层元素向目标元素传播，而事件冒泡是从目标元素向外层传播。事件委托是一种开发技巧，它利用事件冒泡机制，将子元素的事件统一绑定在父元素上，通过 event.target 判断触发源，从而减少事件绑定并提高性能。  

### 区分event.target和event.currentTarget

event.target是事件触发本身的元素，event.currentTarget是绑定事件的元素

stopPropagetion的作用：阻止事件继续冒泡

preventDefault()的作用是阻止浏览器的默认行为

比如a标签点击的默认行为就是跳转地址，用preventDefault就可以阻止跳转，还有form的表单默认提交事件
