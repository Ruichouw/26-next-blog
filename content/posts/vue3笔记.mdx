## vue3和vue2的区别

### 实现响应式的方式

Vue2 使用 `Object.defineProperty`，Vue3 使用 `Proxy`

`Object.defineProperty`的弊端是监听对象的每一个属性，但是新增对象属性，这个新属性是没有被监听的，所以新属性失去响应式，虽然通过 `Vue.$set` 方法来增加 **指定对象指定属性的响应性**

> 响应式是不需要操作dom，数据改变，自动更新页面

### Options API 和 Composition API

`Options`类型的 `API`，数据、方法、计算属性等，是分散在：`data`、`methods`、`computed`中的，若想新增或者修改一个需求，就需要分别修改：`data`、`methods`、`computed`，不便于维护和复用。

` Composition API`可以用函数的方式，更加优雅的组织代码，让相关功能的代码更加有序的组织在一起。

### 生命周期的变化

创建前：beforeCreate -> 使用setup()

创建后：created -> 使用setup()

挂载前：beforeMount -> onBeforeMount

挂载后：mounted -> onMounted

更新前：beforeUpdate -> onBeforeUpdate

更新后：updated -> onUpdated

销毁前：beforeDestroy -> onBeforeUnmount

销毁后：destroyed -> onUnmounted

> 用onMounted 钩子在组件挂载后发送异步请求，这是因为发送异步请求通常需要确保组件已经挂载，以便正确地操作DOM或者更新组件的状态。

## 响应式数据

### ref

- `ref`接收的数据可以是：**基本类型**、**对象类型**。


- 若`ref`接收的是对象类型，内部其实也是调用了`reactive`函数。

`ref`创建的变量必须使用`.value`（模版中可以不用)

ref()返回`RefImpl`对象,`RefImpl`对象的value属性的属性值是对象转换成的 reactive



###reactive

只能接受对象类型，ref接受对象类型实际是内部会把该对象转换为 reactive

### 解构会失去响应式

```javascript
const state = reactive({
  count: 0
})

const { count } = state
```

这里发生了`const count = state.count`,此时count = 0只是一个普通变量

### toRefs

toRefs把 reactive 对象的每个属性都包装成一个 ref

```javascript
let {name,gender} =  toRefs(person)
```

## computed

```javascript
import { computed } from "vue"

const result = computed(() => {
  return expression
})
```

只根据已有响应式数据计算出一个新值，并且具有缓存能力，只有依赖变化才会重新计算，而依赖值是根据返回值自动收集的，`computed()`返回值是 **一个 ref 对象**

`computed()`默认是不能直接修改值的，需要修改值的话需要在定义变量的时候写 **getter + setter**

## watch

`watch(source, callback, options)`是用来监听数据变化时执行某一操作的函数

source 要监听的数据

> 监听的数据是reactive的话，需要用回调函数的形式返回监听数据，不然无法正确追踪
>
> ```javascript
> watch(() => state.count, (newVal, oldVal) => {
>   console.log(newVal)
> })
> ```
>
> 监听多个 reactive 属性通常用 `() => [a, b]`


callback  数据变化执行的回调函数

```
(newVal, oldVal) => {
  console.log(newVal)
 }
```

options  配置项

> immediate 是否立即执行一次，默认是false，不立即执行
>
> deep 是否深度监听对象，即对象内部属性变化也监听

实际场景：

搜索建议（搜索词改变进行数据请求）

分页请求  （监听分页参数变化执行数据请求，分页参数：页码+一页参数多少数据+查询条件）

路由变化请求数据  （URL 参数变了，但组件没有重新创建，需要重新请求数据；详情页切换）

> 详情页切换因为用的是同一个组件，所以不会重新挂载组件，所以数据还是原来的数据，需要通过监听路由变化来重新获取数据
>
> 也可以用key强制更新组件，但是更推荐用watch，因为复用组件性能更好



## watchEffect

自动收集依赖，不需要指定监听源，会立即执行一次

```javascript
watchEffect(() => {
  console.log(count.value)
})
```

## 获取元素实例ref

作用：获取 DOM 元素或组件实例

- 用在普通`DOM`标签上，获取的是`DOM`节点。


- 用在组件标签上，获取的是组件实例对象。可以通过这个引用来调用子组件的方法和数据，但需要子组件`defineExpose`来暴露数据

```javascript
<Person ref="ren"/>

let ren = ref()

  function test(){
    console.log(ren.value.name)
    console.log(ren.value.age)
  }
```

> 组件 ref **只有在组件挂载后才有值**。通常在onMounted()或者nextTick()中调用

## props

用于父组件向子组件传值

子组件定义`props`

```javascript
const props = defineProps({
  title: String,
  count: Number
})

//ts写法
<script setup lang="ts">
interface Props {
  title: string
  count?: number
}

const props = defineProps<Props>()
</script>
```

结合`emit`

```javascript
const emit = defineEmits(['update:count'])

const add = () => {
  emit('update:count', newValue)
}

<Child v-model:count="num" />
//实际为
<Child 
  :count="num" 
  @update:count="num = $event" 
/>
```

```
<Child :count="count" @updateCount="count = $event" />
```

这里的$event指的是emit传过来的修改值，等价于`<Child @updateCount="(value) => count = value" />`

## nextTick()

等 DOM 更新完成之后再执行代码

Vue 为了性能，采用 **异步批量更新机制**（即数据改变多次，合并成1次DOM更新），数据变化不会立即更新 DOM，而是加入更新队列，在当前事件循环结束后统一更新

```
const change = () => {
  count.value++
  console.log(document.querySelector('#num').innerText)
}
```

所以这个例子，count.value++执行了，数据改变，但是还没更新dom，而是推入微任务队列，先执行打印的同步代码，所以打印的还是原来的dom值

```
nextTick(() => {
  // DOM 已更新
})
```

## 路由

### 路由参数

路径参数    查询参数

```
const route = useRoute()
console.log(route.params) // 路径参数
console.log(route.query)  // 查询参数
```

假设路由地址`/user/123?name=tom`

```
route.params.id   // "123"
route.query.name  // "tom"
```

### useRoute和useRouter区别

useRoute用于获取路由参数信息

useRouter用于路由跳转

```javascript
 const router = useRouter()
 router.push('/user/123')
 //用绝对路径，路径必须带 /，不然会成为相对路径直接拼接
```

用name+params的方式跳转

```
router.push({
  name: 'user',
  params: { id: 123 }
})
```

> 注意：不能用path+params的方式跳转，因为 path 是“最终字符串”，不需要再解析 params

### `<router-link>`

```
<router-link to="/user/123">用户页</router-link>
```

replace 返回按钮不会回到上一页

```
<router-link to="/login" replace>登录</router-link>
```

`<router-link>` 在路径匹配当前路由时，会自动添加 `router-link-active` 或 `router-link-exact-active` 类名，用于实现导航高亮。其中前者是模糊匹配，后者是精确匹配

### 路由懒加载

```
const routes = [
  {
    path: '/home',
    component: () => import('../views/Home.vue')
  }
]
```

- `component` 是一个函数，返回 `import()`
- **效果**：当用户访问 `/home` 时，才加载 `Home.vue`

###  路由重定向

redirect作用：将特定的路径，重新定向到已有路由。

## v-model

现“数据的双向绑定”，即数据变 → 页面变；页面变 → 数据也变

```javascript
<input v-model="msg" />
//等价于
<input
  :value="msg"
  @input="msg = $event.target.value"
/>
```

组件中的v-model

```javascript
<MyInput v-model="msg" />
//等价于
<MyInput
  :modelValue="msg"
  @update:modelValue="msg = $event"
/>
```

组件中`v-model="msg"`实际为 `v-model:valueModel="msg"`，valueModel是规定的默认值

也可以自己自定义

```javascript
<MyInput v-model:title="msg" />
```

## 插槽slot

允许父组件往子组件里“塞内容”，即子组件留坑，父组件填内容

### 默认插槽

```javascript
//子组件
<template>
  <div class="box">
    <slot>默认内容</slot>
  </div>
</template>
//父组件
<Child>
  <p>这是插进去的内容</p>
</Child>

//结果
<div class="box">
  <p>这是插进去的内容</p>
</div>
```

### 具名插槽

当子组件有多个位置需要父组件传入内容填入时，用具名插槽来区分不同的位置，也就是给slot组件就加上name属性

父组件用#name来区分，每个位置要用template 包裹

`<template #header>`等价于`<template v-slot:header>`

### 作用域插槽

组件把数据“传给父组件使用”

```javascript
<template>
  <slot :user="user"></slot>
</template>

<Child v-slot="{ user }">
  <p>{{ user.name }} - {{ user.age }}</p>
</Child>
```

## 缓存组件

让组件在切换时不被销毁，而是被缓存起来

用 `<KeepAlive>` 包裹组件

被缓存的组件会新增两个生命周期钩子：

- `**activated**`：组件被激活时触发（重新显示）
- `**deactivated**`：组件被停用时触发（相当于被隐藏时）

include属性可以指定需要缓存的组件



## component 组件

component是动态组件，通过 `:is` 属性决定渲染哪个组件

tab切换

```javascript
<script setup>
import Home from './Home.vue'
import User from './User.vue'
import { ref } from 'vue'

const current = ref('Home')

const components = {
  Home,
  User
}
</script>

<template>
  <button @click="current = 'Home'">Home</button>
  <button @click="current = 'User'">User</button>

  <component :is="components[current]" />
</template>
```
